1.4 MyBatis一级缓存

缓冲针对数据查询（select）有效，一级缓存是针对SqlSession对象设置的。一级缓存不能关闭时强制使用的技术。

对于同一个SqlSession对象，反复执行同一个查询SQL语句的之后，自动启用一级缓存。同一个SQL语句：String sql1 = “”; String sql2 = “” 必须满足 sql1.equals(sql2)=true。一级缓存的原理：当我们执行第一个查询的时候，将查询的结果集存储在SqlSessin内部的容器中，再执行第二个SQL语句的时候，前提是SQL是一样的，SqlSession取出内部存储的数据返回使用，不会查询数据库。

测试中：SQL语句仅仅执行了一次

SqlSession执行一次SQL，然后close()，缓存肯定就没有了

SqlSession执行一次SQL，然后commit()，缓存也没有了

1.5 MyBatis二级缓存

一级缓存针对SqlSession对象的，二级缓存针对的是mapper.xml文件。框架底层会对mapper.xml开启一个缓存，一般情况下不使用的

针对于mapper.xml开启二级缓存的时候，保证User对象可以被序列化

1.1 什么是框架 frame

开发企业级项目的时候才使用框架，小型项目案例基本不使用（大财小用，避免大炮打蚊子）

第1章 Maven概述

1.1 Maven的概念

Maven是项目进行模型抽象，充分运用的面向对象的思想，Maven可以通过一小段描述信息来管理项目的构建，报告和文档的软件项目管理工具。Maven 除了以程序构建能力为特色之外，还提供高级项目管理工具。由于 Maven 的缺省构建规则有较高的可重用性，所以常常用两三行 Maven 构建脚本就可以构建简单的项目。

1.2 Maven的作用

  • maven对项目的第三方构件（jar包）进行统一管理。向工程中加入jar包不要手工从其它地方拷贝，通过maven定义jar包的坐标，自动从maven仓库中去下载到工程中。

  • maven提供一套对项目生命周期管理的标准，开发人员、和测试人员统一使用maven进行项目构建。项目生命周期管理：编译、测试、打包、部署 、运行。

  • maven对工程分模块构建，提高开发效率。

第一章 Filter过滤器

1.1 Filter的概念

现在的生活中到处都是过滤器：比喻坐火车，一般情况下要过安检的。通过2个安检，一个是进站时，一个是进入候车室，进入站台上车。

第一个安检过不去，不会到达第二个安检。第一个过了，第二个没过，依然不能进站上车。安全检查就是过滤器，安全检查可以拦截旅客。

过安检是排在前面的人先安检，先进站，是数据结构中的队列，数据先进先出。

所谓的过滤：就是程序中的一个安检，拦截客户端，运行通过，客户端才能访问到服务器程序

871. 最低加油次数

题目描述：

汽车从起点出发驶向目的地，该目的地位于出发位置东面 target 英里处。

沿途有加油站，每个 station[i] 代表一个加油站，它位于出发位置东面 station[i][0] 英里处，并且有 station[i][1] 升汽油。

假设汽车油箱的容量是无限的，其中最初有 startFuel 升燃料。它每行驶 1 英里就会用掉 1 升汽油。

当汽车到达加油站时，它可能停下来加油，将所有汽油从加油站转移到汽车中。

为了到达目的地，汽车所必要的最低加油次数是多少？如果无法到达目的地，则返回 -1 。

注意：如果汽车到达加油站时剩余燃料为 0，它仍然可以在那里加油。如果汽车到达目的地时剩余燃料为 0，仍然认为它已经到达目的地。
