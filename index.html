第一章 XML

1.1 xml介绍

1.1.1 什么是xml

XML是eXtensible Markup language的缩写，可扩展标记语言。语言的结构就是由标签组成，语法形式和HTML基本一致的。

XML标签是可以自定义的，标签的中属性也是自定义，属性值都是自定义的。

XML是由万维网联盟（W3C）在1998年推出的xml版本的1.0标准，2004年推出1.1版本标准，两个版本根本就不兼容，1.1没人用

1.1.2 xml作用

  • HTML标记语言（5.0），最长久的版本HTML4.01，做网页，浏览器呈现画面。
  • XML扩展标记语言(1.0)，不是做网页，不给用户看，保存数据使用，用于配置文件
  • properties配置文件，配置形式简单，数据结构单一，仅仅适合Java技术开发
  • XML可以提供更加丰富数据形式，灵活的配置结构，适合任何语言开发
    • JavaWeb程序，核心配置文件就是xml
    • 框架Spring，SpringMVC，MyBatis核心配置文件都是xml
    • SpringBoot,SpringCloud 配置 yml

1.2 xml基本语法

  • 标签<标签名> </标签名> 开始标签和结束标签
  • 属性跟随开始标签，属性不同名，属性值必须有””
  • 严格区分大小写

1.3 XML文档组成

1.3.1 文档声明

文档声明，W3C标准：必须写在文件的0行0列位置

1.3.2 标签元素

标签元素:Element，写法形式和HTML一样，自己定义

一个格式良好的xml文档，应该有一个根标签 RootElement

1.3.3 属性和属性值

属性必须写在开始标签里面，多个属性之间空格分开，同一个标签中，属性不能同名，属性必须有值，必须有双引号。

1.4 XML的约束

将来我们主要就是根据XML约束中的规定来编写XML配置文件，而且会在我们编写XML的时候根据约束来提示我们编写, 而XML约束主要包括DTD和Schema两种。

什么是DTD

DTD（Document Type Definition），文档类型定义，用来约束XML文档。规定XML文档中元素的名称，子元素的名称及顺序，元素的属性等。

DTD重点要求

开发中，我们很少自己编写DTD约束文档，通常情况我们都是通过框架提供的DTD约束文档，编写对应的XML文档。常见框架使用DTD约束有：mybatis、struts2、hibernate等。

  • Schema

Schema是新的XML文档约束；
Schema要比DTD强大很多，是DTD 替代者；
Schema本身也是XML文档，但Schema文档的扩展名为xsd，而不是xml。
Schema 功能更强大，数据类型更完善
Schema 支持名称空间

我们将来使用SSM框架中的Spring、SpringMVC框架时，会涉及到一点点对Schema约束的设置。不过不必紧张，有IDEA的支持操作会非常简单，我们现在只需要理解基本概念即可。

首先我们要理解一个概念：名称空间，英文：name space：

1.5 XML解析

读取xml文件就是解析，人们提出了解析思想，如何来读取xml文件

  • dom思想
    • 文档对象模型,dom思想
    • 将整个文档全部读取出，放在内存中，就是一个对象document
    • document对象获取xml的具体内容
    • 好处：整个文档都在内存中，保留了文档结构，可以增删改查
    • 弊端：读取速度慢，占内存
  • sax思想
    • 文档比作大山，推山的方式读取
    • 以事件驱动的方式，读取文件
    • 读取一行，问开发人员，要不要
    • 如果不要数据，继续读取下一行，上一行就没有了
    • 好处：读取速度快，占内存小
    • 弊端：没有文档结构，只能读取
  • 解析器：按照解析思想，实现的程序

1.6 DOM思想解析XML

  • dom4j.jar包中的核心API：
    • SaxReader类：读取xml文件，创建文档对象document
    • Document接口：表示整个xml文档
    • Element接口：表示文档的标签元素

1.7 xpath技术

Xpath技术，基于DOM思想建立，精准定位到你想要的标签，不需要层层读取

第二章 Tomcat服务器

2.1 Tomcat 的简介

Tomcat 服务器是一个开源免费的轻量级Web应用服务器，由Sun公司和Apache软件基金会联合发起的。在中小型系统和并发量小的场合下被普遍使用，是开发和调试Servlet、JSP 程序的首选。也是绝大多数JavaEE开发工程师所必用的

2.2 Tomcat的作用

Apache Tomcat，是一个免费的，Java语言编写，Web应用程序服务器。Tomcat从现在开始，作为我们开发的程序的运行环境，IDEA中编写程序，复制到Tomcat中执行，客户端浏览器可以直接访问我们写的程序。

2.3 Tomcat的安装和启动

apache-tomcat-9.0.63.zip解压缩，任意目录都是可以的，记住位置

启动Tomcat服务器之前，必须配置JAVA_HOME

Tomcat的bin目录下启动文件：startup.bat，开启服务器，占用机器端口8080,等待客户端浏览器访问。

Tomcat页面的位置：tomcat解压目录下/webapps/ROOT/index.jsp

Tomcat的bin目录下关闭文件：shutdown.bat

第三章 HTTP协议

3.1 HTTP协议概念

HTTP协议（超文本传输协议 Hyper Text Transfer Protocol），是开发JavaWeb程序的基石，非常重要。HTTP协议是网络通信模型（TCP/IP）传输层的协议，是TCP协议的子协议。规定的是客户端浏览器和服务器之间的通信规则。HTTPS：HTTP协议的加密传递
