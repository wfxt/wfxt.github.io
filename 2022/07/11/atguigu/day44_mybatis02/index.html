MyBatis框架

第一章 原始dao的开发方式

dao：负责操作数据库，执行SQL语句，返回SQL的结果集，业务层调用即可。

业务层：每个功能的业务逻辑不一样，但是dao层无论是什么业务逻辑，执行SQL语句，能不能不写dao层！MyBatis框架简化dao层的书写，只写接口，永远不写实现类。框架采用动态代理技术来实现不写实现类的效果！

第二章 动态代理开发

2.1 动态代理开发的好处

动态代理的好处：减少代码编写

动态代理的实现原理：MyBatis框架利用了原有JDK中技术（动态代理），可以实现运行期间临时生成接口的实现类！

我们开发人员不需要面对JDK的代理技术，调用MyBatis中的API方法即可。

2.2 动态代理开发规范

  • 要求：SQL语句的配置文件namespace的属性值，必须和接口的全限定名的一致
  • 要求：SQL语句的配置文件中SQL语句标签的id属性值，必须和接口抽象方法的方法名一致
  • 要求：SQl语句的配置文件中SQL 语句标签的参数类型，必须和接口抽象方法的参数一致
  • 要求：SQl语句的配置文件中SQL 语句标签的结果集，必须和接口抽象方法的返回值一致

SqlSession接口方法：getMapper(dao层接口.class) 返回接口的实现类（JDK代理生成）

第四章 手动映射

4.1 什么情况下使用手动映射

当数据表中的列名和pojo中类的属性名不同时，将会出现封装数据失败的现象，MyBatis无法将数据表中的数据准确的封装到pojo对象中，因此必须使用手动映射的方式来实现。

4.2 resultMap标签

为什么需要手动映射配置，如果确实不修改字段名，完全不需要手动映射。

但是：考虑后面的多表查询的问题，数据结果集没有对应的JavaBean对象，只能手动映射结果集

第六章 MyBatis多表查询

6.1 多表关系分析

多表关联：至少两个表关联。分析多表关系的经验技巧：从一条记录出发，不要从表整体去分析，比如分析A表和B表关系，A表中的一条记录对应B表中的几条记录，如果是一条，那么A到B就是一对一；如果多条，那么A到B就是一对多

  • 一对一，从订单表出发，到用户表，是一对一的关系
  • 一对多，从用户表出发，到订单表，一条用户数据可以在订单表中存在多条记录，这就是一对多。通过主外键来体现一对多的表结构。一的一方是主表，多的一方是从表，从表当中有一个字段是外键，指向了主表的主键
  • 多对多，用户和角色表，一个用户可以有很多角色，一个角色有很多用户，多对多通过中间表来体现

6.2 多表关联的SQL语句表达分析

  • 笛卡尔积
    • SELECT *FROM USER,orders
  • 关联查询
    • 内关联 innder join on
      • SELECT * FROM USER u,orders o WHERE u.id=o.user_id
      • SELECT * FROM USER u INNER JOIN orders o ON u.id=o.user_id;
    • 左外连接
      • SELECT * FROM USER u LEFT JOIN orders o ON u.id=o.user_id
    • 右外连接
      • SELECT * FROM USER u RIGHT JOIN orders o ON u.id=o.user_id
