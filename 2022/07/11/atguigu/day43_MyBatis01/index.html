第一章 认识框架

1.1 什么是框架 frame

比喻：框架就是一个搭建好的舞台（灯，音响，烟雾，伴奏…），表演什么，是人的事情，其他的事情都交给框架完成。关注的是基础的技术问题都已经完成了，将框架看成是一个半成品。框架称为拿来主义！

框架：开发人员，不必考虑基础技术问题，专注你的业务功能！

1.2 框架解决什么问题

框架解决的是技术整合问题，软件的规模非常大，开发一个软件从0开始肯定不现实，此时必须需要一个非常优秀的框架出现，将最基本的技术整个完毕，在框架之上开发自己的业务功能，提升开发的效率！

1.3 什么时候使用框架

开发企业级项目的时候才使用框架，小型项目案例基本不使用（大财小用，避免大炮打蚊子）

第二章 MyBatis框架的使用

2.1 MyBatis框架介绍

  • Mybatis原本是Apache软件基金会的一个开源项目叫做iBatis，2010年这个项目由Apache迁移到了google code管理才改名为Mybatis，2013年又迁移到了GitHub。

  • Mybatis是一个优秀的持久层框架（Dao层框架），它是对JDBC的封装，使得开发者只需要关注Sql语句（业务）本身即可，无需开发者处理加载驱动、获取连接、创建Statement等繁琐的过程。

  • Mybatis最大的特点是把Sql语句写在XML配置文件当中。而且Mybatis执行完Sql语句之后可以以对象形式返回（POJO/POJO集合等）。

  • Mybatis是一个实现了ORM思想的持久层框架。

  • ORM：Object/Relation Mapping 对象/关系映射。

  • ORM思想：将数据库中的关系数据表映射为JAVA中的对象，把对数据表的操作转换为对对象的操作，实现面向对象编程。因此ORM的目的是使得开发人员以面向对象的思想来操作数据库。

    比如：原来insert使用的是insert into…，如果使用实现了ORM思想的持久层框架，就可以在Java程序中直接调用api，比如insert(User)，达9到操作对象即操作数据库的效果。Hibernate框架是一个全自动的ORM持久层框架，只需要编写POJO，在xml中定义好Pojo属性和数据表字段的映射/对应关系，就可以在java中实现类似 insert(User)的操作。Sql语句都不用写。但是因为性能等问题，市场占有率越来越低

    Mybatis框架是一个半自动的ORM持久层框架，也可以在Java中实现类似 insert(User)的操作最终操作数据库，但是需要我们自己写Sql语句。Mybatis是目前比较流行的Dao层框架。

1.2 SQL语句中的参数传递-面试题

  • SQL语句中的参数有2个传递方式
    • ${}和#{}，能用#不用$
    • #{} 运行时期转成?，防止注入攻击
    • ${} 运行时期直接拼接字符串
    • ${} 适合于SQL语句中的名称用法，表名，列名
      • select * from user order by ${列名}
