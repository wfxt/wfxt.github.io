Request&Response

第一章 Request对象

1.1 Request概述

Request对象封装客户端浏览器的所有请求数据。请求数据在HTTP协议中分为了三个部分

  • 请求行

  • 请求头

  • 请求体

  • javax.servlet.ServletRequest接口：

    • Sun公司设计接口，做一个与协议无关的接口
    • 客户端和服务器之间基本上都是HTTP协议，设计了一个接口，满足Http协议
    • javax.servlet.http.HttpServletRequest接口：封装客户端的请求参数，适用HTTP协议的
    • Request对象，是接口HttpServletRequest的实现类对象
    • 接口的实现，Tomcat引擎创建，调用请求方法service传递

1.2 Request获取HTTP请求的内容

1.2.1 获取请求行

方法名 方法作用
String getMethod() 获取客户的请求方式，返回字符串
String getRequestURI() 获取客户端请求的服务器的地址，返回字符串
StringBuffer getRequestURL() 获取客户端请求的服务器的地址，返回字符串
String getContextPath() 获取Web应用程序上下文的名字

1.2.2 获取请求头

1.2.3 获取请求参数

方法名 作用
String getParameter() 获取客户端请求参数
String[] getParameterValues() 获取客户端请求参数，一键多值
Map<String,String[]> getParameterMap() 获取客户端所有的请求参数
  • getParameter()
  • getParameterValues()
  • getParameterMap()

1.3 解决获取请求参数乱码

客户端的请求GET方式没有乱码，只有POST请求才有乱码

Tomcat8.5开始，对GET请求进行了编码表的设置，才有UTF-8解码

请求是POST方式，Tomcat将采用另一个编码表解码(iso8859-1)

设置Request对象使用的编码表：方法 setCharacterEncoding(“utf-8”)

1.4 使用BeanUtils将Map中的数据封装到JavaBean对象中

getParameterMap()全部的请求参数都会存储Map集合，使用集合需要遍历，简单做法，将Map集合中的数据，存储到JavaBean对象。

Apache软件基金会：开发工具包BeanUtils.jar包

1.5 请求转发

Servlet接收到客户的请求，将请求转发到另一个Servlet中，由另一个Servlet对客户端做出响应

request对象的方法：getRequestDispatcher() 返回对象，对象称为转发器

  • 请求转发

    浏览器向Servlet1发送请求

    • Servlet1将请求送给Servlet2
    • Servlet2为客户端进行响应

  • 转发是服务器的内部行为，客户的浏览器根本不知道，因此浏览器地址不会改变！

    地址直接写要转发的servlet的请求地址，不需要写web应用程序上下文的名字

1.6 Request请求域对象

Request对象的作用域比较小，ServletContext也是域对象，作用范围是整个应用程序程序。但是request域对象的作用域一次请求是有效的，浏览器进行了第二次请求，域对象的数据消失了

  • setAttribute(String key,Object value)存储键值对
  • Object getAttribute(String key)键获取值

第二章 Response

2.1 Response的概述

Response对象是为客户端浏览器进行响应的对象（数据会出现在浏览器中），HTTP协议中包括：

  • 响应行：状态码 ，协议版本
  • 响应头：键:值
  • 响应体：页面的正文，浏览器解释执行

javax.servlet.ServletResponse接口：Sun公司设计的和协议无关的，响应接口。现在都是HTTP协议，满足HTTP协议，又做了一个接口javax.servlet.http.HttpServletResponse，接口的实现类Tomcat引擎创建

2.2 使用Response设置响应行

Tomcat服务器，能够根据我们程序的执行结果，自动设置状态码，开发人员根本不需要设置状态码！

2.3 使用Response设置响应头

方法 作用
addHeader(String key,String value) 添加响应头键值对
addIntHeader(String key,int value) 添加响应头键值对
addDateHeader(String key,long time) 添加响应头键值对
setHeader(String key,String value) 设置响应头键值对
setIntHeader(String key,int value) 设置响应头键值对
setDateHeader(String key,long time) 设置响应头键值对

add方法，可以设置一键多值，set方法不能重复键，如果键重复，就覆盖以前的值！！

2.4 使用Response设置响应体

2.4.1 Response设置响应体-字符输出流

response对象的方法：getWriter()返回打印流对象– PrintWriter

打印流输出中文，出现乱码

pw.println(“你好浏览器”); 编码表是谁，ISO889-1

浏览器解析文字，编码表是谁，平台默认的GBK

Sun公司在response对象中定义方法setContentType

2.4.2 Response设置响应体-字节输出流

response对象中的字节流和字符流是互斥，只能使用一个

2.5 重定向

浏览器重定向：浏览器向服务器Servlet1发送请求，Servlet1对客户端进行响应，告诉浏览器另一个服务器程序的地址。浏览器主动发起第二次请求，请求Servlet2服务器程序
