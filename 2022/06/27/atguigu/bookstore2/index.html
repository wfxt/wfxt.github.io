第一章 JSP技术

1.1 什么是jsp

jsp全名（Java Server Page）Java服务器端页面，运行在Web服务器上的一种动态网页技术（页面根据用户的需求而改变）。在Tomcat里面运行通过浏览器进行访问。

JSP本质上：Java代码嵌入HTML

1.2 Java代码嵌入HTML

  • 使用特殊的符号：<% 嵌入java代码 %>
  • 使用特殊的符号：<%=向页面中输出结果%>
  • 使用特殊的符号：<%! 嵌入Java代码%>

1.3 jsp的执行原理

所有的jsp文件运行时期，放在Tomcat服务器的work目录下。定义的jsp文件，会变成.java文件，是Tomcat服务器做出改变，javac编译为class文件，JVM执行

源文件-a.jsp，被Tomcat服务器转成了a_jsp.java，继承父类HttpJspBase，继承父类HttpServlet

jsp == servlet

  • 使用特殊的符号：<% 嵌入java代码 %> 放在 a_jsp.java类中的service方法中
  • 使用特殊的符号：<%=向页面中输出结果%> 被放在打印流的输出方法中，浏览器响应
  • 使用特殊的符号：<%! %> 被放在，类的成员变量位置

修改了jsp的源码后，浏览器必须重新请求，tomcat在会重新转换文件

1.4 jsp的内置对象

  • jsp的9大内置对象，已经建立完毕的，直接使用即可
    • request请求对象
    • response响应对象
    • ServletContext application，整个Web应用程序对象，最大域对象
    • ServletConfig config , Servlet的配置对象
    • JspWriter out，打印流对象，向页面中响应数据
    • page，对象是this，当前页面对象
    • HttpSession session，会话对象
    • pageContext 最小域对象，当前页面有效
    • exception 页面中的异常对象

2.1 三层架构

如果不做三层架构形式的拆分：

所有和当前业务功能需求相关的代码全部耦合在一起，如果其中有任何一个部分出现了问题，牵一发而动全身，导致其他无关代码也要进行相应的修改。这样的话代码会非常难以维护。

所以为了提高开发效率，需要对代码进行模块化的拆分。整个项目模块化、组件化程度越高，越容易管理和维护，出现问题更容易排查。

  • 表述层：又可以称之为控制层，负责处理浏览器请求、返回响应、页面调度
  • 业务逻辑层：负责处理业务逻辑，根据业务逻辑把持久化层从数据库查询出来的数据进行运算、组装，封装好后返回给表述层，也可以根据业务功能的需要调用持久化层把数据保存到数据库、修改数据库中的数据、删除数据库中的数据
  • 持久化层：根据上一层的调用对数据库中的数据执行增删改查的操作

为什么要使用base标签统一页面基础访问路径

因为在页面中有很多的a标签、表单以及Ajax请求(以后会学)都需要写访问路径，而在访问路径中项目路径是一样的，所以如果不统一编写项目路径的话，就会发生当项目路径发生改变的时候该页面所有用到项目路径的地方都需要更改的情况

base标签的语法规则
  • base标签要写在head标签内
  • base标签必须写在所有其他有路径的标签的前面
  • base标签使用href属性设置路径的基准
  • base标签生效的机制是：最终的访问地址=base标签href属性设置的基准+具体标签内的路径
  • 如果某个路径想要基于base中的路径进行路径编写，那么它不能以/开头

2.2.3 加密方式介绍

  • 对称加密：加密和解密使用的相同的密钥，常见的对称加密算法有:DES、3DES
  • 非对称加密：加密和解密使用的密钥不同，常见的非对称加密算法有:RSA
    • 加密：使用公钥加密
    • 解密：使用私钥解密
  • 消息摘要: 消息摘要算法的主要特征是加密过程不需要密钥，并且经过加密的数据无法被解密，只有相同的原文经过消息摘要算法之后，才能得到相同的密文，所以消息摘要通常用来校验原文的真伪。常用的消息摘要算法有:MD5、SHA、MAC

我们在书城项目中采用MD5算法对密码进行加密
