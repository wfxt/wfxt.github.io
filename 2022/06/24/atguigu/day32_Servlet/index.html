Servlet服务器程序

第一章 Web项目部署Tomcat

1.1 Web项目配置与访问

项目创建IDEA中，项目运行在Tomcat服务器中！

启动Web项目的按钮：将我们写好的Web项目，复制到Tomcat服务器中，并启动Tomcat服务器，等待客户端浏览器进行访问！！

浏览器地址栏输入地址：http://localhost:8080/web应用程序上下文的名字，回车，会看到Web项目下的index.jsp网页的效果

1.2 发布Tomcat

IDEA需要提速，启动Tomcat的时候，将Tomcat软件读取到内存中，修改Tomcat的配置文件，将项目的执行目录做修改，修改到当前项目的out目录下！！

  • 修改IDEA，将你项目直接放在Tomcat的Webapps下

  • IDEA中的项目和Tomcat的webapps下的项目结构
    • IDEA中
      • Web项目名字：javaweb
      • src源码目录
      • web目录：html，图片,css,javascript
        • web目录虽然在tomcat中没有，但是web目录下的所有内容，都直接发布到了tomcat中
    • webapps中
      • 项目的名字：javaweb
      • src目录是没有的
      • web目录没有
      • WEB-INF目录
        • classes – IDEA中的src下的源码，编译后的class文件
        • lib – 存储使用的第三方jar
        • web.xml 的核心配置文件

1.3 Web资源

存储在服务器中的内容，只要可以被浏览器访问到的，都web资源。

  • 静态资源：固定不变的资源
    • HTML文件，图片，CSS，下载的word
  • 动态资源
    • 网络上的内容：根据用户的输入改变而改变，能够和用户交互

第二章 Servlet

2.1 Servlet的概念

Servlet（Server + Applet）：运行在Web服务器端（Tomcat）的小程序。Servlet的主要作用：接收客户端浏览器的请求，还可以为客户端浏览器做出响应。

  • 学习Servlet技术三大目的：
    • 让程序运行在服务器端
    • 接收客户端浏览器的请求参数
    • 用户需求，对客户端进行数据响应

JavaWeb技术有三大组件：Servlet，Filter，Listener

JavaWeb还有十三规范：Tomcat服务器支持其中的2个Servlet和jsp

2.2 Servlet程序快速入门

  • 定义类实现接口Servlet（5个方法）
  • 通知Tomcat，我们自己写了接口实现类 – web.xml配置
  • 浏览器发起请求，访问我们自定义的Servlet程序

2.3 客户端请求Servlet执行流程

  • 执行流程关键点
    • 写web.xml的配置文件
    • Tomcat引擎通过读取配置文件，反射建立对象
    • Tomcat引擎调用方法service

第三章 Servlet的进阶

3.1 Servlet的生命周期和生命周期方法

对象的生命周期：对象创建的时机，对象调用过程，对象的销毁时机

  • Servlet对象的生命周期：
    • Servlet狭义：专门指的是Servlet接口
    • Servlet广义：接口的所有的实现类
  • Servlet对象的创建时机
    • Tomcat引擎对象Servlet接口实现类的对象
    • 当客户端第一次请求Servlet时创建
    • Tomcat引擎调用方法init()，只调用一次，客户端每次请求Servlet，都不会在调用init

  • Tomcat服务器启动的时候，直接创建Servlet对象

    • web.xml配置标签

    • 标签必须配置在servlet标签里面

    • <load-on-startup>10</load-on-startup>

目录方式和后缀名方式，不能同是配置

  • 扩展小点:

  • Tomcat的解压目录conf

    • 配置文件web.xml （全局）

      • 当前Tomcat中所有Web项目，都共用的配置文件

    • 我们自己写的项目，也有配置文件web.xml

      • 当前项目自己使用的

    • 当前全局的配置文件，和项目自己的配置文件写的一样，按照自己写的为准

  • 全局web.xml配置

3.3 ServletConfig对象

  • javax.servlet.ServletConfig接口
    • 表示当前Servlet对象配置的接口
    • Servlet接口实现类的方法init中，存在对象
  • 接口的方法：
    • String getServletName()返回当前的Servlet的名字
    • getInitParameter() 获取Servlet的初始化参数：web.xml配置的
  • web.xml配置
  • 接口的方法：
    • ServletContext getServletContext() 返回，获取的是ServletContext 接口的实现类对象
    • ServletContext 接口代表整个Web应用程序对象

3.5 Servlet的体系结构

  • 接口Servlet是Sun公司的标准接口：实现接口的类，都可以放在Web服务器中执行，被浏览器直接进行访问 （service方法）
  • HTTP协议：Sun为了满足这个协议的要求，单独开发了一个Servlet接口实现类HttpServlet
  • 我们自己定义Servlet的时候，无需直接实现该接口，继承HttpServlet

3.6 使用Idea直接创建Servlet

IDEA自己帮我们写web.xml配置，只是写了一半，另一半servlet-mapping自己写

  • MyServlet类是不是接口Servlet的实现类 （肯定）
  • Servlet接口，service方法，是Sun的标准，Tomcat服务器调用方法–service
  • HttpServlet源码

结论：原理不用先搞明白，继承方式，请求调用doGet，或者doPost就可以了

    • 接口实现类，Tomcat引擎创建
    • 一个Web应用程序中，对象是唯一的

3.5 域对象

  • ServletContext是域对象
    • 是容器，可以存储键值对 == Map集合
    • 作用范围最大：整个Web应用程序都可以使用
  • 域对象都会有三个通用方法
    • void setAttribute(String key,Object value) 域对象中存储键值对
    • Object getAttribute(String key)返回域对象中，键对应的值，没有此键，返回null
    • void removeAttribute(String key) 删除域对象中的键值对

第四章 Servlet注解开发

注解取代web.xml配置文件

原理：讲过注解解析，反射获取注解的属性值

Tomcat技术包扫描，扫描所有的包中的所有的类，反射获取每个类上，是否存在注解@WebServlet，如果存在注解，获取注解属性urlPatterns（浏览器的访问地址）

注解和配置文件，不能并存
