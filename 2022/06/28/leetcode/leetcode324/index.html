方法一：排序

首先，我们可以很容易想到一种简单的解法：将数组进行排序，然后从中间位置进行等分（如果数组长度为奇数，则将中间的元素分到前面），然后将两个数组进行穿插。参考HexCat的题解

例如：
对于数组[1, 5, 2, 4, 3]，我们将其排序，得到[1, 2, 3, 4, 5]，然后将其分割为[1, 2, 3]和[4, 5]，对两个数组进行穿插，得到[1, 4, 2, 5, 3]。

但是这一解法有一个问题，例如，对于数组[1, 2, 2, 3]，按照这种做法求得的结果仍为[1, 2, 2, 3]。如果题目不要求各元素严格大于或小于相邻元素，即，只要求nums[0] <= nums[1] >= nums[2] <= nums[3]...，那么这一解法是符合要求的，但题目要求元素相互严格大于或小于，那么需要稍微做一点改进。

为了方便阅读，我们在下文中定义较小的子数组为数组A，较大的子数组为数组B。显然，出现上述现象是因为nums中存在重复元素。实际上，由于穿插之后，相邻元素必来自不同子数组，所以A或B内部出现重复元素是不会出现上述现象的。所以，出现上述情况其实是因为数组A和数组B出现了相同元素，我们用r来表示这一元素。而且我们可以很容易发现，如果A和B都存在r，那么r一定是A的最大值，B的最小值，这意味着r一定出现在A的尾部，B的头部。其实，如果这一数字的个数较少，不会出现这一现象，只有当这一数字个数达到原数组元素总数的一半，才会在穿插后的出现在相邻位置。以下举几个例子进行形象地说明：

例如，对于数组[1,1,2,2,3,3],分割为[1,1,2]和[2,3,3]，虽然A和B都出现了2，但穿插后为[1,2,1,3,2,3]，满足要求。
而如果2的个数再多一些，即[1,1,2,2,2,3]，分割为[1,1,2]和[2,2,3]，最终结果为[1,2,1,2,2,3]，来自A的2和来自B的2出现在了相邻位置。

出现这一问题是因为重复数在A和B中的位置决定的，因为r在A尾部，B头部，所以如果r个数太多（大于等于(length(nums) + 1)/2），就可能在穿插后相邻。要解决这一问题，我们需要使A的r和B的r在穿插后尽可能分开。一种可行的办法是将A和B反序：

例如，对于数组[1,1,2,2,2,3]，分割为[1,1,2]和[2,2,3]，分别反序后得到[2, 1, 1]和[3, 2, 2]，此时2在A头部，B尾部，穿插后就不会发生相邻了。

当然，这只能解决r的个数等于(length(nums) + 1)/2的情况，如果r的个数大于(length(nums) + 1)/2，还是会出现相邻。但实际上，这种情况是不存在有效解的，也就是说，这种数组对于本题来说是非法的。

对于mid为什么选做(n + 1) / 2而不是 n / 2，我们可以做如下考虑。若mid = n / 2，则当n = 1时，i = -1，数组越界。

实际上，A、B数组个数必须相同或相差1.当n为偶数时，如[0,1,2,3,4,5]，两边各占一半。当n为奇数时，如[0,1,2,3,4,5,6]，最好为A数组元素略多更好。当mid = (n + 1) / 2时有A = [4,3,2,1,0]，B = [8,7,6,5],穿插为[4,8,3,7,2,6,1,5,0]。若mid=n/2，则有A = [3,2,1,0]，B = [8,7,6,5,4]，按照先A后B的穿插为[3,8,2,7,1,6,0,5,4]，不符合题意，若先B后A的穿插，则为[8,3,7,2,6,1,5,0,4]，因此还需要一次额外的反转。所以，为了代码书写的便捷，我们考虑设mid = (n + 1) / 2，且mid元素在B数组中。

复杂度分析：

  • 时间复杂度：进行了排序和一次遍历，复杂度为O(nlogn)
  • 空间复杂度：需要对元素组进行一次拷贝，复杂度为O(n)

方法二：三向切分（3-way-partition）

在上一解中复杂度为O(nlogn)因为用到了排序，但我们发现，我们实际上对A、B内部的顺序并不感兴趣，只有满足上面所述的大小及长度关系即可。因此我们可以采用快速选择法找到中间数，具体可以参考215. 数组中的第K个最大元素中的快速排序的选择算法，该算法可以在O(n)内完成排序。

复杂度分析：

  • 时间复杂度：O(n)，其中 n 为数组的长度。找到数组中排序为 k 的数需要的时间复杂度为O(n)，对数组进行三向切分需要的时间复杂度为 O(n)，对数组进行摆放的时间的复杂度为 O(n)，总的时间复杂度为O(n)。
  • 空间复杂度：因为对数组进行了拷贝，因此复杂度为O(n)。

方法三：索引转换

此解法为方法一或者方法二的基础上的优化（数学组大胜利）。假设当前数组进行 partition 后的顺序为：
nums[0],⋯,nums[i]⋯,nums[mid],⋯,nums[i],⋯,nums[n−1]

设数组长度 n ，令 x = (n + 1) / 2，我们分为两种情况来讨论：

当 n 为偶数时，此时我们已知摆动排序后的顺序为：
nums[mid]<nums[n−1]>nums[mid−1]⋯nums[1]<nums[mid+2]>nums[0]<nums[mid+1]

我们可以观察到原数组的元素索引 [0,1,2,⋯,n−1] 在新数组中的索引分别为：
n−2,n−4,⋯,0,n−1,n−3,⋯,1

根据观察我们可以发现，如下转换关系：

当原数组的索引 i<x 时：其在新数组中对应的索引为*n−2×(i+1)*。

当 i≥x 时：其在新数组中对应的索引为n−2×(i−x)−1=n−2×i+2×x−1=n−2×(i+1)+n+1。

我们可以将上述式子进行合并可以得到：对于原数组的第 i 个元素，在新数组中对应的元素索引为(n+1)(n−2×(i+1)+n+1)mod(n+1)=(2×n−2×i−1)mod(n+1)

当 n 为奇数时，此时我们已知摆动排序后的顺序为：
nums[mid]<nums[n−1]>nums[mid−1]⋯nums[1]<nums[mid+1]>nums[0]

我们可以观察到原数组的元素索引[0,1,2,⋯,n−1] 在新数组中的索引分别为：
n−1,n−3,⋯,0,n−2,n−4,⋯,1

根据观察我们可以发现如下转换关系：

当原数组的索引 i<x 时：其在新数组中对应的索引为 n−1−2×i

当 i≥x 时：其在新数组中对应的索引为n−2×(i−x)−2=n−2×i+2×x−2=n−1−2×i+n。

我们可以将上述式子进行合并可以得到：对于原数组的第 i个元素，在新数组中对应的元素索引为(n-1-2*i+n)%n=(2*n-2*i-1)%n

综上我们可以对上述进行合并，当 n 为偶数时，(2×n−2×i−1) 需要对 n + 1 进行取模，我们可以将索引i的转换进行合并为(2×n−2×i−1)%(n∣1)，此时我们直接对摆动排序后的数组进行 partition 操作即可，我们按照 partition 后的数组顺序进行遍历，然后依次进行比较交换即可，这样就避免了再次将数组进行拷贝复制。
